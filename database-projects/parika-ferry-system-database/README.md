# Parika Ferry System Database

## Overview

The Parika Ferry System Database is a relational database design project created to support ferry operations between Parika, Supenaam, and Bartica. The system models vessels, terminals, routes, schedules, bookings, passengers, crew members, tickets, and vehicles.

## Purpose

The purpose of this project is to demonstrate database design, relational modelling, normalization, and SQL reporting for a realistic transportation management system.

## Key Features

- Vessel and terminal management
- Directional ferry route modelling
- Sailing schedules
- Passenger bookings and ticketing
- Vehicle classification and fare tracking
- Crew assignment tracking
- SQL reports for revenue, bookings, vessels, vehicles, and schedules

## Database Design Concepts Used

- Entity Relationship Diagram (ERD)
- Relational schema design
- Functional dependencies
- First Normal Form (1NF)
- Second Normal Form (2NF)
- Third Normal Form (3NF)
- Primary keys and foreign keys
- Weak entity modelling
- SQL joins, aggregation, filtering, and subqueries

## Technologies Used

- SQL
- MySQL / MariaDB
- ERD Design
- Database Normalization
- Relational Database Design

## What I Learned

This project helped me understand how to move from a real-world problem to a structured database design. It also strengthened my understanding of normalization, relationships, weak entities, and writing SQL queries for operational reporting.
