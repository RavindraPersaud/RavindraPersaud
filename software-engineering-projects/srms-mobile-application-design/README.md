# Student Records Management System (SRMS) Mobile Application Design

## Overview

This project focuses on the analysis and design of a proposed mobile application for the University of Guyana's Student Records Management System (SRMS). The purpose of the project was to improve student access to academic services through a mobile-friendly system that could support course registration, application tracking, timetable viewing, academic records, notifications, and selected payment-related features.

This was a Software Engineering project, so the focus was on project planning, requirements analysis, system design, UML modelling, architecture, and system testing rather than building a production-ready mobile application.

## Purpose

The purpose of this project was to apply Software Engineering principles to a real-world university system. The project explored how a mobile SRMS could improve access, usability, and system interaction for students while still integrating with the existing SRMS web platform and backend infrastructure.

## My Contribution

My work contributed to the analysis and design of the proposed system, including project planning, systems analysis, UML modelling, technical documentation, and system testing artefacts.

## Key Features Covered

* Student login and access to academic services
* Course registration workflow
* Timetable viewing
* Academic record and GPA viewing
* University application process for prospective students
* Document upload process
* Payment handling for foreign/CARICOM students
* Push notifications for updates
* Integration with the existing SRMS backend
